無痛升級部落客 殺手級整合 HEXO NexT theme 與 Github 部署

發表於 2020-03-22 更新於 2020-03-25 分類於 hexo Disqus：

當會使用 Makeup Language 來進行文件的書寫後，
利用hexo 平台產生網頁並 deploy 到 Github Page 的整合，讓您以最快速的方式開始寫作博客。

利用 Hexo 架站 除存在 github Page 中

  • Hexo 架站與基本架構
    • hexo theme.schema “NexT.Pisces” v5.1.4
  • Github Page (github.io) HTML Page & version control
    • integration with hexo

Version

hexo: 4.2.0
hexo-cli: 3.1.0

create ROOT directory

先在 Documents 下建立 Github 路徑，作為 Github 的保存路徑

安裝 hexo command line tools: by using node.js (node.js should be installed in your computer)

安裝 hexo

Initial your blog name

完成hexo 安裝，接著安裝 Hexo 的幾個基本的頁面：

About Page

產生 About 關於我 網頁

修改 source/about/index.md 的內容

Categories Page

產生 Categories 類別 網頁

修改 source/categories/index.md

tags Page

產生 tags 標籤 網頁

修改 source/tags/index.md

安裝 hexo theme

註冊 guthub.com 帳號 目前使用 hawkchou14

在 github.com 註冊完成後，新增一個 repository 名稱為 hawkchou14.github.io (後面一定要加 .github.io, 作為 github PAGE 識別使用)

安裝完成後，修改 _config.yml “Hexo 基本架構參數”

修改 theme 相關參數

很多與 Theme 相關的參數需要根據需求調整，須小心調整相關參數

修改檔案 theme/next/_config.yml

此時會將 .md 檔案 render 成為 html 的檔案放置於 ./public 目錄中

清除快取資料

有時產生的資料會快取 (Cache) 在伺服器內，因此會將目前生成的資料清除後重建
清除的指令

將會刪除 ./public 目錄，然後重新生成(generate)內容

進行 Local 測試

會開啟 http://localhost:4000/ 的伺服器指向到 local 的 ./public 路徑的 web server.
可以開啟瀏覽器，使用 URL: http://localhost:4000/ 來確認產生的內容。

部署到 Github

此時可以利用瀏覽器開啟 URL: [https://hawkchou14.github.io/] 來進行瀏覽。
